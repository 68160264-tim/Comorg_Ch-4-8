บทที่ 4 หน่วยความจำแคช (Cache Memory)
4.1 ภาพรวมของระบบหน่วยความจำคอมพิวเตอร์ (Computer Memory System Overview)

ระบบคอมพิวเตอร์สมัยใหม่ประกอบด้วยหน่วยความจำหลายระดับ (Memory Hierarchy) เพื่อให้สามารถตอบสนองต่อความต้องการด้าน ความเร็ว ความจุ และต้นทุน ได้อย่างเหมาะสม

โดยทั่วไป หน่วยความจำที่มีความเร็วสูงมักจะมีราคาสูงและมีความจุน้อย ในขณะที่หน่วยความจำที่มีความจุมากมักจะมีความเร็วต่ำกว่า ดังนั้นสถาปัตยกรรมคอมพิวเตอร์จึงใช้แนวคิด ลำดับชั้นของหน่วยความจำ (Memory Hierarchy) เพื่อให้ระบบสามารถทำงานได้อย่างมีประสิทธิภาพ

ลำดับชั้นของหน่วยความจำโดยทั่วไปประกอบด้วย

Registers

Cache Memory

Main Memory (RAM)

Secondary Storage เช่น SSD หรือ Hard Disk

ข้อมูลที่ถูกใช้งานบ่อยจะถูกเก็บไว้ในหน่วยความจำที่มีความเร็วสูงกว่า เช่น Cache เพื่อให้โปรเซสเซอร์สามารถเข้าถึงได้อย่างรวดเร็ว

แนวคิดสำคัญที่ทำให้ระบบนี้ทำงานได้คือ หลักการของ Locality

1. Temporal Locality

ถ้าข้อมูลหนึ่งถูกใช้งานในขณะหนึ่ง มีแนวโน้มสูงที่จะถูกใช้งานอีกครั้งในอนาคตอันใกล้

2. Spatial Locality

ถ้าตำแหน่งหน่วยความจำหนึ่งถูกเข้าถึง มีแนวโน้มว่าตำแหน่งใกล้เคียงจะถูกเข้าถึงในเวลาใกล้กัน

แนวคิดเหล่านี้เป็นพื้นฐานสำคัญของการออกแบบ Cache Memory

4.2 หลักการทำงานของ Cache Memory

Cache Memory เป็นหน่วยความจำขนาดเล็กที่มีความเร็วสูงมาก ซึ่งถูกติดตั้งอยู่ระหว่าง CPU และ Main Memory

จุดประสงค์หลักของ Cache คือ

ลดเวลาในการเข้าถึงข้อมูลของ CPU

ลดจำนวนครั้งที่ต้องเข้าถึงหน่วยความจำหลัก

เมื่อ CPU ต้องการอ่านข้อมูล จะมีขั้นตอนดังนี้

CPU ตรวจสอบข้อมูลใน Cache ก่อน

ถ้าพบข้อมูล → เรียกว่า Cache Hit

ถ้าไม่พบ → เรียกว่า Cache Miss

ระบบจะดึงข้อมูลจาก Main Memory มาเก็บไว้ใน Cache แล้วส่งให้ CPU

เนื่องจากโปรแกรมส่วนใหญ่มักใช้ข้อมูลซ้ำ ๆ หรือใช้ข้อมูลที่อยู่ใกล้กัน Cache จึงช่วยเพิ่มประสิทธิภาพของระบบได้อย่างมาก

4.3 องค์ประกอบของการออกแบบ Cache (Elements of Cache Design)

การออกแบบ Cache จำเป็นต้องพิจารณาปัจจัยสำคัญหลายประการ ได้แก่

1 ขนาดของ Cache (Cache Size)

Cache ที่มีขนาดใหญ่สามารถเก็บข้อมูลได้มากขึ้น ทำให้มีโอกาสเกิด Cache Hit สูงขึ้น

อย่างไรก็ตาม Cache ที่มีขนาดใหญ่ก็มีข้อเสีย เช่น

ต้นทุนสูง

เวลาเข้าถึงอาจเพิ่มขึ้น

ดังนั้นผู้ออกแบบระบบต้องหาสมดุลระหว่าง ขนาดและประสิทธิภาพ

2 รูปแบบการจัดเก็บข้อมูล (Mapping Function)

Mapping Function คือวิธีการกำหนดว่าข้อมูลจาก Main Memory จะถูกเก็บไว้ที่ตำแหน่งใดใน Cache

มีรูปแบบหลัก ๆ ดังนี้

Direct Mapping

Main Memory แต่ละบล็อกสามารถถูกเก็บได้เพียงตำแหน่งเดียวใน Cache

ข้อดี

โครงสร้างง่าย

เข้าถึงเร็ว

ข้อเสีย

มีโอกาสเกิด Cache Miss บ่อย

Fully Associative Mapping

บล็อกข้อมูลจาก Main Memory สามารถถูกเก็บในตำแหน่งใดก็ได้ใน Cache

ข้อดี

ลดปัญหา Cache Miss

ข้อเสีย

ต้องใช้ฮาร์ดแวร์ที่ซับซ้อน

Set Associative Mapping

เป็นการผสมระหว่าง Direct Mapping และ Fully Associative

Cache จะถูกแบ่งเป็นหลาย Set และข้อมูลสามารถถูกเก็บในตำแหน่งใดก็ได้ภายใน Set นั้น

วิธีนี้เป็นรูปแบบที่นิยมใช้มากในคอมพิวเตอร์สมัยใหม่

4.4 อัลกอริทึมการแทนที่ข้อมูล (Replacement Algorithms)

เมื่อ Cache เต็มและต้องนำข้อมูลใหม่เข้ามา จำเป็นต้องเลือกข้อมูลบางส่วนออกจาก Cache

อัลกอริทึมที่ใช้กันบ่อย ได้แก่

LRU (Least Recently Used)

ลบข้อมูลที่ไม่ได้ถูกใช้งานมานานที่สุด

ข้อดี

มีประสิทธิภาพสูง

ข้อเสีย

การติดตามข้อมูลอาจซับซ้อน

FIFO (First In First Out)

ข้อมูลที่ถูกนำเข้ามาก่อนจะถูกนำออกก่อน

ข้อดี

โครงสร้างง่าย

ข้อเสีย

อาจไม่เหมาะสมกับรูปแบบการใช้งานบางประเภท

Random Replacement

เลือกข้อมูลแบบสุ่มเพื่อแทนที่

ข้อดี

ใช้งานง่าย

ข้อเสีย

ประสิทธิภาพไม่แน่นอน

4.5 นโยบายการเขียนข้อมูล (Write Policy)

เมื่อ CPU ต้องการเขียนข้อมูลลงหน่วยความจำ Cache ต้องกำหนดว่าจะจัดการกับ Main Memory อย่างไร

Write Through

ข้อมูลจะถูกเขียนไปยัง Cache และ Main Memory พร้อมกัน

ข้อดี

ข้อมูลมีความสอดคล้องกันเสมอ

ข้อเสีย

มีการเข้าถึง Main Memory บ่อย

Write Back

ข้อมูลจะถูกเขียนเฉพาะใน Cache ก่อน และจะถูกเขียนกลับไปยัง Main Memory เมื่อถูกแทนที่

ข้อดี

ลดจำนวนการเขียนไปยัง Main Memory

ข้อเสีย

การควบคุมซับซ้อนกว่า

4.6 Cache ในระบบคอมพิวเตอร์สมัยใหม่

ในโปรเซสเซอร์สมัยใหม่ มักมี Cache หลายระดับ ได้แก่

L1 Cache

อยู่ใกล้ CPU มากที่สุด

ความเร็วสูงมาก

ขนาดเล็ก

L2 Cache

มีขนาดใหญ่กว่า

ความเร็วรองลงมา

L3 Cache

ใช้ร่วมกันระหว่างหลาย Core
